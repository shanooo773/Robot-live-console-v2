┌─────────────────────────────────────────────────────────────────────────────┐
│  ERROR SCENARIOS                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

1. Login before email confirmation:
   POST /auth/login → 403 Forbidden
   "Email not confirmed. Please check your inbox."

2. Expired confirmation token:
   GET /auth/confirm?token=expired → 400 Bad Request
   "Confirmation token has expired. Please request a new confirmation email."

3. Invalid confirmation token:
   GET /auth/confirm?token=invalid → 400 Bad Request
   "Invalid confirmation token. Please check the link..."

4. Invalid Google token:
   POST /auth/google → 401 Unauthorized
   "Invalid Google token"

5. Email already exists:
   POST /auth/register → 400 Bad Request
   "Email already exists"

┌─────────────────────────────────────────────────────────────────────────────┐
│  KEY SECURITY FEATURES                                                       │
└─────────────────────────────────────────────────────────────────────────────┘

✅ Confirmation tokens:
   - Generated with itsdangerous.URLSafeTimedSerializer
   - Signed with SECRET_KEY (same as JWT)
   - 1-hour expiry (configurable)
   - Cannot be forged or tampered with

✅ Google OAuth:
   - ID token verified server-side with Google API
   - Verified against GOOGLE_CLIENT_ID
   - Prevents token substitution attacks
   - No client-side validation

✅ Password security:
   - SHA-256 hashing with unique salt per user
   - Salt stored with hash (format: "salt:hash")
   - Password never stored in plain text

✅ Email validation:
   - Required before account activation
   - Prevents spam/bot registrations
   - Validates user ownership of email

✅ Demo users:
   - Bypass email confirmation for testing
   - Negative user IDs to avoid conflicts
   - Environment-configured credentials
